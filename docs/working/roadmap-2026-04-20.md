# Dendra Roadmap — What's Next

**Generated:** 2026-04-20.
**Status snapshot:** Library 156 tests green. Patent + marketing
ready. Public-facing plumbing (repo, PyPI, landing page, analyzer,
Cloud) not yet shipped.

**Framing principle:** ship the **priority-date-locking** moves
first (provisional patent before any public disclosure), then the
**launch-ready** moves (PyPI, landing page, analyzer), then the
**revenue-enabling** moves (Cloud MVP, first domain pack, design
partners), then the **paper-strengthening** moves. Do NOT pull
order forward — priority-date loss is the only unrecoverable
mistake.

---

## Critical path (must do in this order)

These gate everything else. Cannot parallelize.

1. **File provisional patent.** (Day 0, ~$75 + 1 day work.)
   Package is ready in `patent/`. Register USPTO Patent Center
   account, upload, pay, save the application number.
2. **Snapshot provenance artifacts** per `patent/05-provenance-
   snapshot.md`. (Same day, 15 min.)
3. **Publish to PyPI** — `v0.2.0` release. (Day 1, ~half a day.)
4. **Create public GitHub repo** `github.com/axiom-labs-os/dendra`.
   Transfer commits from the current worktree. (Day 1, ~half
   a day.)
5. **arXiv preprint** of the "When Should a Rule Learn?" paper.
   (Day 1-2, assumes paper is complete.)

Once these five land, priority and public existence are both
secured. Everything after can parallelize.

## Workstream 1 — Public-launch readiness (weeks 1-3)

**Goal:** `pip install dendra && dendra analyze ./my-repo` works
end-to-end for any developer who hears about us on Hacker News.

### 1.1 Landing page — `dendra.dev`

- Single-page site. Hero: the Figure 1 transition curves.
- One code block: the `@ml_switch` decorator example.
- "Install with pip" button. No marketing copy above the fold.
- Pricing table collapsed below the fold (use the
  `docs/marketing/dendra-one-pager.md` table).
- Link to arXiv, GitHub, docs, patent-pending badge.
- Estimated effort: 2-4 days (Astro/Next.js static site).

### 1.2 Documentation site — `dendra.dev/docs` or ReadTheDocs

- Auto-generated API reference from docstrings.
- Quickstart (copy from `README.md`).
- 3-5 tutorials:
  - "Wrap your first classifier"
  - "Graduate from Rule to LLM-shadow"
  - "Use the `dendra roi` reporter"
  - "Integrate with Claude Code via the SKILL"
  - "Phase 4 for safety-critical classifiers"
- Estimated effort: 3-5 days using Sphinx or MkDocs.

### 1.3 `dendra init` CLI

- `dendra init ./my_file.py:my_function` emits a minimal diff
  wrapping the target function with `@ml_switch`.
- Agent-first install path — Claude Code / Cursor can invoke
  this non-interactively without hallucinating.
- Estimated effort: 2-3 days. Concrete, well-scoped.

### 1.4 SKILL.md for Claude Code + Cursor rules

- `docs/integrations/SKILL.md` per the operational-dx design.
- `.cursor/rules/dendra.md` with the same content adapted to
  Cursor's context surface.
- Submit a PR to Anthropic's skill registry (if one exists).
- Estimated effort: 1 day.

### 1.5 Example gallery

- `examples/` directory with 5-7 runnable `.py` files:
  - `01_hello_world.py` — wrap a trivial rule
  - `02_phase_0_outcome_log.py` — record outcomes, read back
  - `03_phase_1_llm_shadow.py` — add an LLM observer
  - `04_phase_4_safety_critical.py` — `safety_critical=True`
  - `05_benchmark_atis.py` — reproduce Figure 1 for ATIS
  - `06_output_safety_gate.py` — LLM output classification
  - `07_roi_report.py` — generate a ROI report
- Each <100 lines, runnable with `python examples/NN_*.py`.
- Estimated effort: 2 days.

### 1.6 GitHub Actions CI

- `.github/workflows/test.yml`: Python 3.10, 3.11, 3.12, 3.13, 3.14.
- Run full test suite + ruff lint.
- Coverage reporting via Codecov.
- Automatic release to PyPI on tagged push.
- Estimated effort: 1 day.

### 1.7 CONTRIBUTING.md + issue/PR templates

- How to file an issue, how to propose a new pattern, how to
  contribute a domain pack.
- GitHub issue templates for bug / feature / pattern.
- Estimated effort: half a day.

**Workstream 1 total estimated effort:** ~3 weeks founder-only
or ~1.5 weeks with a part-time contractor on landing page + docs.

## Workstream 2 — Static analyzer MVP (weeks 2-6)

**Goal:** `dendra analyze ./my-repo` produces a useful report.
This is the Snyk-style lead-generation engine.

### 2.1 AST walker + pattern library v1

- 15-20 initial patterns covering the most common classifier
  shapes (§6.3 of the patent spec + `business-model-and-moat.md`
  §2.1):
  - P1 if-elif-else string dispatchers
  - P2 match-case dispatchers
  - P3 dict-lookup dispatchers
  - P4 keyword scanners
  - P5 regex dispatchers
  - P6 LLM-prompted classifiers
- Each pattern returns: file path, line range, inferred label
  set, inferred label cardinality, regime classification,
  fit-score.
- Estimated effort: 1 week.

### 2.2 CLI and JSON output

- `dendra analyze <path>` → stdout summary + optional `--json` file.
- Diffable against prior runs for CI.
- Estimated effort: 2 days.

### 2.3 GitHub Action for `dendra analyze`

- One-line YAML users drop into their `.github/workflows/`
  repos. Comments on PRs with "this PR added 2 classification
  sites; Dendra-fit score 4/5."
- Estimated effort: 2 days.

### 2.4 Public analyzer-corpus badge

- Each run on a public repo anonymously increments a
  corpus-pattern counter. The more public runs we get, the
  sharper our patterns become (§4 moat in
  `business-model-and-moat.md`).
- Opt-out flag explicit and honored.
- Estimated effort: 3 days (includes backend for
  corpus-pattern tracking).

**Workstream 2 total:** ~2-3 weeks.

## Workstream 3 — Paper strengthening (weeks 2-6, parallel)

**Goal:** Tier-A items complete before NeurIPS submission
deadline.

Ordered by expected paper-uplift (from
`docs/papers/2026-when-should-a-rule-learn/results/strengthening-plan.md`):

### 3.1 Multi-seed training-order validation (A2)

- Run each benchmark 5× with shuffled training streams.
- Report transition depth as mean ± stddev.
- Estimated effort: 1 day (mostly wall-clock compute).

### 3.2 Per-example McNemar paired-test on all 4 benchmarks (A1)

- We shipped the plumbing, only ran ATIS. Re-run Banking77,
  CLINC150, HWU64 with the full paired-test apparatus.
- Estimated effort: 2 hours compute + 1 day analysis.

### 3.3 Stronger ML head (A3)

- `SentenceTransformerHead` — wraps a sentence-transformer
  embedding + a logistic head.
- Re-run all 4 benchmarks; plot old-head vs new-head
  transition curves on the same axes.
- Expected result: ML ceiling rises, transition depth shifts
  earlier.
- Estimated effort: 3 days.

### 3.4 Hand-authored rules for ATIS (A4)

- Author a careful keyword rule for ATIS by inspecting first
  100 training examples manually.
- Compare against the auto-generated rule side-by-side.
- Defuses the "your rule is a straw-man" critique.
- Estimated effort: 1 day.

### 3.5 Phase 1 LLM-shadow with Claude Haiku / GPT-4o-mini (B5)

- API budget: ~$50-100.
- Run Phase 1 on all 4 benchmarks with a competent frontier
  LLM.
- Expected: LLM trivially beats rule; may or may not beat
  ML head.
- Critical — turns §9.3 from future-work into headline result.
- Estimated effort: 2 days + wall-clock inference.

### 3.6 Fixed seeds + pinned dataset versions (A5)

- Pin HF dataset revision hashes.
- `numpy.random.seed()` at runner entry.
- `reproduce.sh` script.
- Estimated effort: half a day.

### 3.7 arXiv submission

- Write up the paper from the outline (~20-30 pages).
- Estimated effort: 1-2 weeks of writing.

**Workstream 3 total:** ~3-4 weeks including writing.

## Workstream 4 — Cloud MVP + first domain pack (weeks 4-12)

**Goal:** paying customer by end of quarter.

### 4.1 Cloud MVP scope

Minimum viable hosted product:
- **Auth** — Google/GitHub OAuth.
- **Workspaces** — org/team abstraction.
- **Switch registration** — users register their switches, get a
  token to authenticate outcome writes.
- **Hosted outcome log** — writes to managed storage (S3 + a
  metadata DB — Postgres or DynamoDB).
- **Dashboard** — one page per switch showing transition curve,
  phase, outcome-count, last-seen.
- **Stripe integration** — Free / Solo / Team / Pro / Scale
  tiers from `business-model-and-moat.md` §3.1. Self-serve
  checkout.
- Estimated effort: **6-10 weeks** for founder solo; 4-6 weeks
  with a contractor.

### 4.2 Dynamic analyzer ("Insight" tier)

- Measurement-only wrapper per `business-model-and-moat.md`
  §2.2 — customers run it in their codebase to capture
  traffic shape, output distribution, optional outcome signal.
- Bundle into Cloud Pro+ tier; standalone $49/mo as noted in
  pricing.
- Estimated effort: 2 weeks.

### 4.3 Domain pack #1 — support-ticket triage

- Pre-trained ML head on a labeled support-ticket corpus
  (synthetic or derived from a public dataset like Banking77).
- Rule template.
- Evaluation corpus with hold-out.
- Packaged as `pip install dendra-pack-support-triage`.
- Priced at $1,500 one-time or $150/mo bundled with Cloud Pro.
- Estimated effort: 2-3 weeks (most of the time is corpus
  curation).

### 4.4 Design-partner program

- Landing page describing the program.
- Application form.
- 3-5 invitees. Contracts at $10-25k for 6-month priority
  access + case-study rights.
- Concretely: reach out to specific engineering teams known to
  have classifier problems. Linear, PostHog, Supabase, Cal.com,
  and our internal Axiom classroom deployment are the natural
  targets.
- Estimated effort: ongoing over Q1-Q2.

**Workstream 4 total:** ~10-12 weeks. This is where the
bootstrap revenue plan lives.

## Explicit "NOT doing" list

Things we deliberately defer, with reasons:

- **Multi-language implementations** (Rust, Go, JS). Python-first
  until there's documented pull; listed as within-scope in the
  patent so priority is preserved.
- **Federation-GA with MPC / DP-federated-learning.** Too complex
  for Y1; risks enablement objections on the patent. Ship basic
  federated outcome-pattern aggregation only.
- **Certifications beyond SOC 2 Type 1.** Each cert is $30-150k
  and weeks of work. Only pursue when a specific named customer
  blocks without it.
- **Advanced analyzer features** — IDE plugins, LSP, deep
  integrations. Ship the static scanner first; extend when real
  customer usage pulls.
- **A hosted-Dendra marketplace for third-party domain packs.**
  Year 3+ idea. Kills focus now.
- **SDR team / outbound sales.** Explicit year-1 exclusion per
  `entry-with-end-in-mind.md` §4.
- **Paper-2 (federation + production case studies).** Year-3
  submission target. Focus on Paper-1 first.
